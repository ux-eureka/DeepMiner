# DeepMiner 产品需求文档 (PRD)

**版本**: 1.0.0
**更新日期**: 2026-02-28
**状态**: 已实现 (Released)

## 1. 产品概述

### 1.1 产品定位
DeepMiner 是一款面向 B 端/C 端产品经理和业务分析师的**动态 AI 业务顾问**。它不仅仅是一个聊天机器人，而是一个具备“裁判能力”和“流程引导能力”的智能助手。通过预设的专家级业务剧本（Playbook），它引导用户一步步剥离表象，挖掘出业务的核心痛点与物理真相。

### 1.2 核心价值
*   **拒绝伪智能**：不同于通用 AI 的泛泛而谈，DeepMiner 拥有严格的“状态路由”机制，用户回答不合格会触发拦截和追问。
*   **结构化产出**：将非结构化的对话过程，转化为结构化的业务洞察报告。
*   **动态流程**：支持无限扩展的业务阶段，根据对话进展动态生成新的探索节点。

---

## 2. 用户角色与流程 (User Flow)

### 2.1 用户画像
*   **B端产品经理**：需要梳理复杂的业务流转、合规限制和岗位操作。
*   **C端产品运营**：关注用户欲望、流失节点和商业变现逻辑。

### 2.2 核心路径
1.  **模式选择**：用户进入首页，选择“B端业务流转”或“C端流量变现”模式。
2.  **动态问询**：
    *   AI 基于当前阶段任务（Phase Task）提问。
    *   用户输入回答。
    *   **核心判定**：AI 裁判判定回答质量。
        *   *Pass*：进入下一阶段，时间轴推进。
        *   *Fail*：停留在当前阶段，AI 辛辣点评并追问细节。
3.  **可视化反馈**：右侧悬浮时间轴实时显示当前进度，已完成节点亮起。
4.  **报告生成**：全流程结束后，系统自动生成结构化的业务洞察 PDF 报告。

## 3. 功能模块详情

### 3.1 核心对话区 (Chat Area)
*   **布局**：极简风格，左侧为对话流，底部为输入框。
*   **消息类型**：
    *   **User Message**：用户输入的内容，右侧气泡。
    *   **System Message (AI)**：AI 的回复，左侧气泡。包含“诊断反馈”和“下一步提问”两部分，Markdown 格式渲染。
    *   **System Warning**：红色/黄色提示，用于系统错误或拦截警告。
*   **交互细节**：
    *   输入框支持自动高度适应。
    *   支持 Markdown 渲染（列表、加粗）。
    *   打字机效果输出 AI 回复。

### 3.2 模式选择器 (Mode Selector)
*   **入口**：输入框左上方的浮动胶囊按钮。
*   **交互**：
    *   点击展开下拉菜单（向上弹出）。
    *   **悬浮预览**：鼠标悬停在模式选项上时，右侧出现“眼睛”图标。
    *   **预览弹窗**：点击眼睛图标，弹出模态框展示该模式的完整 Prompt 剧本，支持一键复制。
*   **数据源**：`src/config/promptsConfig.js` 中的 `MODE_CONFIG`。

### 3.3 动态时间轴 (Floating Stepper)
*   **位置**：屏幕右侧边缘悬浮。
*   **交互**：
    *   默认收起为小圆点，鼠标悬停展开详细文字。
    *   **展开动画**：文字从右向左平滑滑出，保持右对齐布局。
*   **状态**：
    *   *Pending*：灰色空心圆点。
    *   *Active*：绿色实心圆点，带呼吸光环。
    *   *Completed*：绿色实心圆点，带对勾图标。
*   **虚拟化**：使用 `VirtualStepperList` 实现，支持 100+ 节点的无限滚动，仅渲染视口内元素，保障性能。

### 3.4 智能设置面板 (Settings)
*   **入口**：右下角齿轮图标。
*   **功能**：
    *   **API 配置**：配置 Base URL (支持 SiliconFlow 等) 和 API Key。
    *   **环境隔离**：优先读取 `.env` 环境变量，用户手动修改后覆盖默认值。
    *   **安全性**：API Key 仅在本地存储，且敏感信息在持久化时加密。

### 3.5 报告生成器 (Report Modal)
*   **触发**：流程结束（Last Phase Passed）时自动弹出。
*   **内容**：
    *   按 Phase 组织的结构化对话记录。
    *   支持导出 PDF 功能。
*   **容错**：支持动态 Phase 标题渲染，防止数据缺失导致崩溃。

## 4. 技术架构与逻辑 (Tech Specs)

### 4.1 对话引擎 (DeepMiner Engine)
*   **核心 Hook**：`useDeepMinerEngine.ts`
*   **状态管理**：
    *   `currentPhase`: 当前所处阶段索引。
    *   `phases`: 动态阶段定义（支持运行时扩展）。
    *   `history`: 完整的会话历史栈。
*   **LLM 交互协议**：
    *   **Input**：`System Rules` + `History` + `User Input` + `Next Task`。
    *   **Output**：严格 JSON 格式 `{ diagnosis, question, is_passed }`。
    *   **拦截逻辑**：前端解析 JSON，根据 `is_passed` 决定是否 `currentPhase++`。

### 4.2 视觉规范 (Visual Design System)
*   **色彩体系**：
    *   Primary: Emerald Green (`text-emerald-500`, `bg-emerald-50`) - 象征洞察与通过。
    *   Neutral: Zinc (`text-zinc-900`, `bg-zinc-50`) - 极简背景与文字。
    *   Accent: Blue (用于链接与选中态)。
*   **字体排印**：
    *   正文：系统默认 Sans-serif，强调阅读舒适度。
    *   代码/Prompt：Monospace。
*   **动效**：
    *   所有弹窗支持 `zoom-in` / `fade-in`。
    *   侧边栏支持平滑过渡 (`transition-all duration-300`)。
